# RAG Agent 面试问题清单

本文档整理 AI Agent / RAG 方向面试中可能被追问的问题，用于后续复习和补充项目化回答。它不是本项目的需求文档，也不作为开发任务来源；如果某个问题需要转化为本项目能力改造，应按 [改动需求文档沉淀流程](change-documentation-workflow.md) 新建 OpenSpec change。

## 使用方式

- 先用本项目现有实现回答能落地的部分，例如 `ingest -> search -> chat`、Chroma 索引、manifest 增量索引、强制检索、引用生成。
- 对本项目暂未实现的能力，例如 BM25、父子索引、rerank、长期记忆、复杂工具调用、安全白名单，要明确说“当前项目没有实现”，再给可扩展方案。
- 后续如果补充答案，建议按“项目现状 -> 为什么这么做 -> 如果要升级怎么做 -> 风险和验证方式”展开。

## RAG 架构与检索流程

1. 为什么引入父子索引？
2. 为什么在检索阶段引入 BM25？
3. BM25 和向量检索是怎样组合的？比例是如何设置的？
4. 整体检索流程是怎样的？从 query 到最终上下文的完整流程是什么？
5. 检索阶段有没有做 rerank？使用的是什么方式？

## Rerank 与检索结果处理

1. rerank 后一般返回几个块？
2. 为什么选择这个数量？有没有做过验证？
3. rerank 后的 topK 截断是怎么做的？
4. 为什么是这个值？有没有尝试过其他策略？
5. 如果上下文长度不够或过长，你是怎么处理的？

## 上下文工程与记忆机制

1. 讲一下上下文工程是怎么设计的。
2. 上下文拼接的结构是怎样的？
3. 如何避免上下文过长导致模型性能下降？
4. 记忆机制是怎么做的？
5. 短期记忆和长期记忆是如何区分和存储的？
6. 记忆更新策略是什么？

## Agent 设计与工具调用

2. Agent 的任务规划是怎么做的？
3. 规划是由模型完成还是通过规则实现？
4. 多工具调用时如何决定调用顺序？
5. 如果工具调用失败如何处理？

## 安全与防护机制

3. 工具调用的安全控制是怎么实现的？
4. 如何限制模型调用敏感接口？
5. 是否有权限控制或白名单机制？
